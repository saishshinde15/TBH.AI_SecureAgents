This directory contains comprehensive examples demonstrating the key features of the TBH Secure Agents framework, including security profiles and guardrails.

## Comprehensive Example

The `comprehensive_example.py` file demonstrates a complete workflow using both security profiles and guardrails to create a secure multi-agent system that performs a complex security assessment task.

### Overview

The example creates a squad of three specialized security experts:

1. **Security Analyst**: Conducts a security assessment of a cloud-based healthcare system (high security profile)
2. **Security Architect**: Develops mitigation strategies based on the assessment (medium security profile)
3. **Technical Writer**: Creates a comprehensive security report (PII protection profile)

The operations are executed sequentially, with each expert building on the work of the previous one, while maintaining security controls throughout the process.

### Key Features Demonstrated

This example showcases the most important features of the TBH Secure Agents framework:

- **Expert Security Profiles**: Each expert has a specific security profile that controls their behavior
- **Guardrails**: Dynamic inputs control expert behavior at runtime
- **Template Variables**: Variables in expert profiles and operation instructions
- **Conditional Formatting**: Select syntax for conditional content based on guardrail values
- **Secure Context Passing**: Results are securely passed between operations with validation
- **Data Leakage Prevention**: Security profiles prevent sensitive information from being exposed
- **Structured Output Formats**: Clear output specifications reduce the risk of hallucinations

### Running the Example

1. Set your Google API key as an environment variable:
   ```bash
   export GOOGLE_API_KEY=your_api_key_here
   ```

2. Run the example:
   ```bash
   python comprehensive_example.py
   ```

3. Review the output, which will show:
   - The step-by-step process of creating experts with security profiles
   - The definition of operations with template variables
   - The guardrail inputs that control expert behavior
   - The assembly and deployment of the squad
   - The final security report generated by the squad

The output is also saved to `comprehensive_example_output.txt` for easy reference.

## Original Example

The `main_example.py` file is the original example that demonstrates basic security profiles without guardrails.

## Customization

You can modify these examples to create your own secure multi-agent systems by:

1. Changing the experts' specialties and security profiles
2. Modifying the operations' instructions and template variables
3. Adjusting the guardrail inputs to control expert behavior
4. Adding more experts and operations to the squad
5. Changing the security level of the squad

## API Key Security

These examples use environment variables for API keys. In a production environment, you should:

1. Use your own API key
2. Store it securely (e.g., as an environment variable or in a secure vault)
3. Never hardcode API keys in your source code
