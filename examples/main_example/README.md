This example demonstrates a complete workflow using the TBH Secure Agents framework to create a secure multi-agent system that performs a complex cybersecurity task.

## Overview

The example creates a squad of three specialized security experts:

1. **Security Researcher**: Researches the latest trends in ransomware attacks targeting healthcare organizations
2. **Vulnerability Analyst**: Analyzes potential vulnerabilities based on the research findings
3. **Security Report Writer**: Creates a comprehensive security report based on the research and analysis

The operations are executed sequentially, with each expert building on the work of the previous one, while maintaining security controls throughout the process.

## Security Features Demonstrated

This example showcases several key security features of the TBH Secure Agents framework:

- **Expert Security Profiles**: Each expert has a specific security profile that controls their behavior
- **Secure Context Passing**: Results are securely passed between operations with validation
- **Data Leakage Prevention**: High-security profiles prevent sensitive information from being exposed
- **Structured Output Formats**: Clear output specifications reduce the risk of hallucinations

## Running the Example

1. Ensure you have installed the TBH Secure Agents package:
   ```bash
   pip install tbh-secure-agents
   ```

2. Run the example:
   ```bash
   python main_example.py
   ```

3. Review the output, which will show:
   - The process of creating experts with security profiles
   - The definition of operations with clear instructions
   - The assembly and deployment of the squad
   - The final security report generated by the squad

## API Key

This example uses a provided API key for demonstration purposes. In a production environment, you should:

1. Use your own API key
2. Store it securely (e.g., as an environment variable)
3. Never hardcode API keys in your source code

## Customization

You can modify this example to create your own secure multi-agent systems by:

1. Changing the experts' specialties and security profiles
2. Modifying the operations' instructions and output formats
3. Adding more experts and operations to the squad
4. Changing the process type (e.g., from "sequential" to another supported process)
